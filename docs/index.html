The meuh programming language

Introduction

The goal of this project is to implement a programming language which can permit any programmer to work easier without ruminating bad injury about used language.
On the second hand, this can be a hobby for a time.

Kind of language

The language adopts the philosophy of write once, compile anywhere and KISS.
It is a general purpose Turing Complete language with a least those paradigms :
  • Imperative
  • Object-Oriented
  • Procedural
  • Concurrent
  • Generic Programming
  • Reflection
The next features will also be implemented :
  • Thread multiplexing with LLVM coroutines like in go with the goroutine
  • Anonymous functions also called as Lambda expressions
  • ... to be continued

Source code example

package io.meuh.test;

import meuh.lang.Number;
import meuh.lang.Integer;

public class IntegerCalculator extends Calculator implement PolynomialResolver {
  public int main(const String[] &argv) throws PolynomialException {
    if (argv.length < 1) {
      System.stdout.println("Need arguments");
      return 0;
    }
    return resolve(new Number[]{
      Integer.decodeString(argv[1]),
      Integer.decodeString(argv[2]),
    }).toInt();
  }

  protected Number resolve(const Number[] &numbers) throws PolynomialException {
    if (numbers.length != 2) {
      throw new IllegalArgumentException("Only 2 numbers are expected, not: " + numbers.length);
    }
    return resolve(numbers[0], numbers[1]);
  }

  /** Solve a*x+b=0 */
  public Integer polynome(const Integer &a, const Integer &b) throws PolynomialException {
    if (a == 0) {
      throw new PolynomialException("The value of a is zero");
    }
    return -b/a;
  }
}

The toolchain

The compiler front-end will be part of this project, and for the back-end LLVM will be used.
Compilation phases per compilation unit :
1. Lexical Analyzer
Produce tokens
2. Syntax Analyzer
Produce an Abstract Syntax Tree by the usage of an hand written Recursive Descent Parser
3. Semantic Analyzer
Check languages rules.
4. SSA Converter
Convert the AST tree into a SSA tree.
5. LLVMifier
Convert the SSA tree into LLVM assembly language.
6. llvm-as
Translate LLVM assembly language into LLVM bitcode.
7. llvm-link
Link all compilation units together
8. opt
Run global optimization
9. llc
Compile optimized LLVM bytecode into a native object file
10. linker
Use the target platform to link the native object file into an executable binary
Optionally, the 9th and 10th step can be replaced with lli to directly execute program from LLVM bitcode.
